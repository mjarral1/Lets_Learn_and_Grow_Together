Let’s follow their journey as they learn how to track changes, resolve conflicts, and keep their workflow smooth.

A Day in the Life of the Dev Team

Task 1: Checking the Status

Mary started her day by making some changes to the project’s HTML file to improve the header design. She wanted to ensure her changes were properly tracked before committing them. She ran the following command: 

 git status

"Hey team, I see the header file I edited is showing up as modified. Let me stage it!" said Mary. Mary staged her changes: 
git add header.html

Then, she committed them: 
git commit -m "Updated header design"

Mary recalled in her mind that git statushelps check which files are modified, staged, or untracked, making it easier to manage changes.

Task 2: Checking the Project History

Jane wanted to see all the commits made so far in the project. She used the following command: 

git log --oneline

"Wow, this shows a clean summary of our commits!" Jane exclaimed. She noticed that Mary’s "Updated header design" commit was the latest. Jane then ran: 
git log
to see the detailed commit history with author names and timestamps. She murmured that the 
git log
command provides insights into the project's progress and who made changes.
