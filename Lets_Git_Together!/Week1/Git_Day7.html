Lesson 7: Git Collaboration: Building A Git Data Dictionary!

Week: 1

Understand basic Git operations: setting up, creating, and committing in repositories, with a focus on tracking changes and basic branching.

Day: 7

For Lesson 7, we will review our learning of Week 1 and work on a mini project. We will build a data dictionary for Git and GitHub usage.

Review of Lesson 1 to Lesson 6:

In previous lessons, we have covered how to set up Git and GitHub accounts, as well as how to initialize repositories. We also learned some basic Git commands like git add, git commit, git push, and more. Additionally, we explored how to create, switch, merge, push, pull, and remove branches in a Git repository. We also have learned how to review and revert unwanted changes and how to collaborate with other teams with in a shared repository of a project. We also learned about tags and marking milestones of the project.

The Power of Team Collaboration

By Day 7 of their Git learning journey, Mary, Jane, and Raja were growing confident in their Git and GitHub skills.
However, the senior software developer challenged them to consolidate everything they had learned so far by working on a mini project together.
The goal: Create a collaborative Git Commands Data Dictionary repository.

This exercise would simulate a real-world software development scenario, teaching them how to use commands from Day 1 to Day 7 cohesively, resolve potential errors, and create a shared repository that others could benefit from.

“Let’s see if you can combine your knowledge of Git commands and GitHub workflows to create something meaningful,” said the senior developer.

Creating the Data Dictionary


Task 1: Setting Up the Repository

The team decided that Jane, being the most experienced, would create the repository and set up the initial structure.
Jane created a new repository on GitHub called git-commands-dictionary.
git init git-commands-dictionary
cd git-commands-dictionary
She added a README.md file with a project description: 

# Git Commands Data Dictionary 

collaborative repository documenting Git commands and their uses.

Jane made the first commit and pushed it to the remote repository:
git add README.md
git commit -m "Initial commit: Added README with project description"
git branch -M main
git remote add origin GitHub-repo-URL
git push -u origin main
Task 2: Branching for Collaboration

The mentor explained: “Each of you will work on a different section of the data dictionary using separate branches.
This avoids conflicts and promotes clean collaboration.”
Mary created a branch for basic Git commands:
git checkout -b basic-commands
Raja created a branch for advanced Git commands:
git checkout -b advanced-commands
Jane created a branch for GitHub workflows:
git checkout -b github-workflows
Each member added a new Markdown file for their respective sections.

Task 3: Committing and Pushing Changes

Each team member worked on their branches, adding commands to their respective files:
Mary:
- Edited basic-commands.md to document commands like git init, git status, and git add.
- Staged and committed her changes:
git add basic-commands.md
git commit -m "Added documentation for basic Git commands"
- Pushed her branch to GitHub:
git push origin basic-commands
Raja:
- Edited advanced-commands.md to cover commands like git rebase and git cherry-pick.
-Staged, committed, and pushed his branch.
Jane:
-Edited github-workflows.md to explain GitHub actions like pull requests and releases.
-Staged, committed, and pushed her branch.

Task 4: Pull Requests and Reviews

The senior softwared developer guided the team to create pull requests and review each other’s work.
Mary created a pull request for her basic-commands branch on GitHub.
Jane reviewed the changes, suggesting an edit to improve the explanation of git status .
Mary implemented the suggestions, committed the changes, and updated the pull request:
git add basic-commands.md
git commit -m "Updated git status explanation"
git push origin basic-commands
Raja and Jane also followed the same process for their branches.

Task 5: Merging to Main

After resolving all feedback, the mentor instructed the team to merge the branches into the main branch.
Jane merged the pull requests for all branches:
git checkout main
git merge basic-commands
git merge advanced-commands
git merge github-workflows
git push origin main

Task 6: Resolving Merge Conflicts

During the merge, Raja encountered a conflict in README.md With guidance from the senior developer, he resolved it:
1. Opened the conflicting file.
2. Fixed the conflict by keeping the correct lines.
3. Added the resolved file:
git add README.md
4. Committed the fix:
git commit -m "Resolved merge conflict in README.md"

Task 7: Creating a Release

Once all changes were merged, the mentor asked the team to create a GitHub release for version v1.0:
1. Navigate to the Releases tab on GitHub.
2. Click Draft a new release.
3. Select the main branch and tag version v1.0.
4. Add release notes summarizing the commands included in the data dictionary.
5. Publish the release.

Key Takeaways:
1. Git commands can be combined to manage collaborative workflows effectively.
2. Branching allows parallel development, while pull requests enable code reviews.
3. Resolving conflicts is a crucial skill for teamwork.
4. Creating releases on GitHub is a great way to mark project milestones.
