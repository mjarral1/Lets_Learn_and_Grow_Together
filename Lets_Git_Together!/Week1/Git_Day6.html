The Milestone Celebration

After successfully mastering forks, pull requests, and upstream syncing in Lesson 5, the senior developer gathered Mary, Jane, and Raja for another critical learning session.
This time, the team would explore how to use tags and releases in Git to mark important milestones in their project.
The senior developer began explanation with a story:
“Imagine you’re working on software, and after weeks of coding, you finally finish version 1.0.
You want to freeze that exact state of the project for everyone to reference in the future.
That’s where Git tags and releases come into play.”
Mary, Jane, and Raja were eager to learn how they could use these tools to become even better at managing their projects.

Freezing Time with Tags and Releases


Task 1: Understanding Tags

The senior developer explained:
“Tags are like labels you attach to specific commits in your project.
They’re perfect for marking key milestones like software releases or significant updates.”
“Cool!” said Jane. “So it’s like bookmarking an important page in a book?”
“Exactly!” replied the mentor. “Let’s create one together.”

Task 2: Creating a Lightweight Tag

The team decided to mark the milestone for completing their first feature.
Raja used the following command to create a lightweight tag:
git tag v1.0
“Done!” Raja exclaimed.
Mary asked, “How do we share this tag with others?”
The developer replied, “Push the tag to the remote repository.”
Raja pushed the tag:
git push origin v1.0

Task 3: Annotated Tags for Detailed Milestones

The senior developer then introduced annotated tags, which allow adding descriptions to the tag:
“Let’s create a detailed tag for our release,” said the developer.
Jane created an annotated tag:
git tag -a v1.1 -m "Release version 1.1: Added user login feature"
git push origin v1.1
“Now, anyone can see the details of this version by typing git show v1.1 ,” said Jane proudly.

Task 4: Viewing and Deleting Tags

Mary was curious: “What if I accidentally create the wrong tag?”
The mentor reassured her:
“You can delete tags locally and remotely.”
Mary tried:
git tag -d v1.0
git push origin --delete v1.0

Task 5:Publishing Releases on GitHub

The team moved on to GitHub, where the senior developer explained how to create releases:
1. Navigate to the Releases tab in the repository.
2. Click Draft a new release.
3. Select the tag version (e.g., v1.1), add a title, and write release notes.
4. Click Publish Release.
Jane commented, “Releases make it easier to share updates with the world, including compiled binaries or release notes.”

Task 6: Resolving Errors in Tagging

As they practiced, Raja accidentally created a tag on the wrong commit. Jane helped him fix it:
1. Delete the incorrect tag:
git tag -d v1.2
2. Create a new tag on the correct commit:
git tag v1.2 correct-commit-hash git push origin v1.2
