1. Create and Switch Branches:

Branches in Git are like “work zones” where you can build new features without affecting the main project.Imagine you’re writing a book with a friend.
You both want to add chapters, make edits, and explore new ideas — but if you’re both working in the same document, it’s easy to step on each other’s toes. This is where branches come in.

In Git, branches are like separate workspaces where each person can work independently on different parts of a project. You can make changes, experiment with new features, and try out ideas without affecting anyone else’s work.
Once your changes are polished and ready, you can merge them back into the main project, combining everyone’s contributions smoothly.

Imagine you’re building a website with two teammates: Mary and Jane. Mary is designing the homepage, Jane is creating a contact form, and you’re working on the navigation bar.

Here’s how branches keep things organized:
Step 1: Each person creates their own branch. Mary has a homepage branch, Jane has a contact-form branch, and you create a navbar branch.
Step 2: You all work independently, making changes to your own parts.
Step 3: Once everyone is done, you can merge each branch back into the main project, putting all the pieces together.

No one’s work interferes with anyone else’s, and the main project remains stable until you’re ready to merge everything.

To Create a new branch use: 
git branch branchName
To switch to the new branch use: 
 git checkout branchName
Note: Replace branchName with your choice of branch name.

2. Commit changes in different branches.:

Commits record changes to your branch, like saving versions of your work.

git add fileName
git commit -m "Add description of the changes"

1.Let’s say you create or edit a file named index.html.
2.You stage the file with git add index.html
3.You commit the change with: git commit -m "Add index.html with initial content" .
4.Now this change is saved in branchName but not in the main branch.

3. Merge Branches:

Once you’ve completed work on a branch, it’s time to bring it into the main project.
Merging brings the changes from one branch into another (typically from a branchName into the main branch).
1.First, switch to main branch with: git checkout main
2.Now Merge branchName changes into main by running git merge branchName
3.Your main branch now has all the updates from branchName

4. Push Branches to a GitHub Repository:

Pushing uploads your branch (and all its commits) to GitHub, making it available online for others to see or review.
1.To Push your branch to GitHub git push -u origin branchName
Now branchName is available on GitHub, and others can see the changes there.

5. Create and manage pull requests on GitHub:

A pull request (PR) is a request to merge your branch into the main branch on GitHub. It allows team members to review and approve changes before they are merged. 1. Go to GitHub and navigate to your repository.
2.Click the button to Compare & pull request for branchName. 3.Write a short description (e.g., "Adds initial content for index.html").
4.Click Create pull request. 5.Once approved, you can Merge the pull request to add branchName changes to main.

6.Remove a branch:

Use this command to delete a branch that you no longer need. Note that you must not be on the branch you are trying to delete. git branch -d yourBranchName
1.If you have a branch named branchName that you want to delete, type: git branch -d branchName
2.This will delete branchName from your local repository.
If Git prevents deletion because the branch hasn't been merged yet, you can force-delete it with: git branch -D branchName
Note:Use this carefully, as force deletion will remove the branch and its changes permanently from your local repo.

7. Basic Commands:

Initialize a Repository: git init

Add Files to Staging: git add <file-name>

Commit Changes: git commit -m "Initial commit"

View Commit History:git log

Create a new branch: git branch branchName

Switch to a branch: git checkout branchName

Merge a branch: git merge branchName

Remove a branch:git branch -d branchName

Push a branch to GitHub: git push -u origin branchName
