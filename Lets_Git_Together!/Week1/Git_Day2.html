Lesson 2: Branching and Merging in Git!

Week: 1

Understand basic Git operations: setting up, creating, and committing in repositories, with a focus on tracking changes and basic branching.

Overview of Lesson 2:

First Review Lesson 1 content that was about focusing on Git setup, initializing repositories, and basic Git commands like git add, git commit, and git push.

However, Lesson 2 covers how to create, switch, merge, push, and pull branches in git repository.

1. Create and Switch Branches:

Branches in Git are like “work zones” where you can build new features without affecting the main project.Imagine you’re writing a book with a friend.
You both want to add chapters, make edits, and explore new ideas — but if you’re both working in the same document, it’s easy to step on each other’s toes. This is where branches come in.

In Git, branches are like separate workspaces where each person can work independently on different parts of a project. You can make changes, experiment with new features, and try out ideas without affecting anyone else’s work.
Once your changes are polished and ready, you can merge them back into the main project, combining everyone’s contributions smoothly.

Imagine you’re building a website with two teammates: Alice and Bob. Alice is designing the homepage, Bob is creating a contact form, and you’re working on the navigation bar.

Here’s how branches keep things organized:
Step 1: Each person creates their own branch. Alice has a homepage branch, Bob has a contact-form branch, and you create a navbar branch.
Step 2: You all work independently, making changes to your own parts.
Step 3: Once everyone is done, you can merge each branch back into the main project, putting all the pieces together.

No one’s work interferes with anyone else’s, and the main project remains stable until you’re ready to merge everything.

3. Basic Commands:

Initialize a Repository: git init

Add Files to Staging: git add <file-name>

Commit Changes: git commit -m "Initial commit"

View Commit History:git log

Create a new branch: git branch branchName

Switch to a branch: git checkout branchName

Merge a branch: git merge branchName

Push a branch to GitHub: git push -u origin branchName
