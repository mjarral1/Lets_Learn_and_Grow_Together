Overview of Lesson 1:

Lesson 1 covers the Introduction to Git, shows us the Git Setup process, and makes us to learn some Git Basics to begin our journey of Git together!

1. What is Git?

Definition:
  Git is a distributed version control system that helps track code changes, collaborate with others, and manage project history.

Why Use Git?:
  Git allows multiple developers to work on a project simultaneously, maintains a complete history of all changes, and provides tools to merge and resolve code differences.

2. Git Setup:

Installation:
  Download and install Git from git-scm.com.
  Open your command prompt and type the command git. If git is installed properly then it will not show any errors. However, it will show some useful git commands.

Configuration:
  Set up your name and email (Git requires these for commit history).

   git config --global user.name "Your Name"
   git config --global user.email "your.email@example.com"

Please write your name and your email address that you used to make a git account.

3. Basic Commands:

Initialize a Repository: Creates a new Git repository in your project folder.
  git init
Add Files to Staging: To prepare files to be committed, use:
  git add

Please type the name of file you want to create instead of file-name.


Commit Changes: To save changes to the repository with a message, use:
  git commit -m "Initial commit"
View Commit History: To view all previous commits, use:
  git log
